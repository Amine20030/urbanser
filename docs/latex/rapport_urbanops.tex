% --- Résumé ---
\chapter*{Résumé}
\addcontentsline{toc}{chapter}{Résumé}
\thispagestyle{plain}
\begin{resumebox}
\justifying
UrbanOps est une plateforme \textbf{full stack} pour le signalement et le suivi d’\textbf{incidents urbains} (voirie, eau, éclairage, déchets, sécurité, etc.). Le backend \textbf{Spring Boot~3} expose une API REST sous \texttt{/api/v1}, sécurisée par \textbf{JWT}, avec persistance \textbf{PostgreSQL} via \textbf{JPA/Hibernate}. Un module d’\textbf{analyse textuelle} s’appuie sur \textbf{Google Gemini} (réponse JSON) avec \textbf{repli par règles} si l’API est indisponible. Le frontend \textbf{Next.js~14} propose landing, tableau de bord, carte \textbf{Leaflet} et formulaires. Le rapport couvre besoins, conception \textbf{UML}, implémentation, tests \textbf{JUnit~5}, couverture \textbf{JaCoCo} et pilotage qualité \textbf{SonarQube}.
\end{resumebox}

\vspace{0.9cm}
\begin{center}
\textbf{Mots-clés :} ville intelligente ; Spring Boot ; Next.js ; PostgreSQL ; JWT ; Gemini ; JUnit~5 ; JaCoCo ; SonarQube.
\end{center}

% --- Abstract ---
\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
\thispagestyle{plain}
\begin{center}
\begin{minipage}{0.88\textwidth}
\justifying
\textit{UrbanOps is a full-stack smart-city supervision platform for incident reporting and monitoring in an urban context. The backend relies on Spring Boot~3, REST APIs, JWT, and JPA over PostgreSQL. AI-assisted triage uses the Google Gemini API with a deterministic fallback. The Next.js~14 frontend provides dashboards and maps. This document presents requirements, UML design, implementation, testing, and quality gates including JaCoCo and SonarQube.}
\end{minipage}
\end{center}

\vspace{0.9cm}
\begin{center}
\textbf{Keywords:} smart city; Spring Boot; Next.js; PostgreSQL; JWT; Gemini; JUnit~5; JaCoCo; SonarQube.
\end{center}

\ChapterRoman{Cadre général et problématique}
\label{chap:I}
\begin{center}
\begin{minipage}{0.92\textwidth}
\justifying
Ce chapitre situe le \textbf{contexte métropolitain} (Marrakech), la \textbf{problématique} (signalements dispersés, délais de traitement, priorisation) et les \textbf{objectifs} d’UrbanOps : centraliser, tracer, alerter et offrir une interface citoyenne moderne.
\end{minipage}
\end{center}

\SecRoman{Contexte et enjeux}
La transformation numérique des services urbains impose des canaux \textbf{unifiés} pour la remontée d’information et une \textbf{traçabilité} des traitements.

\SecRoman{Objectifs du projet}
\begin{itemize}
  \item Permettre la \textbf{déclaration} d’incidents géolocalisés avec médias ;
  \item Offrir un \textbf{tableau de bord} et des \textbf{alertes} ciblées vers les autorités ;
  \item Garantir une \textbf{piste d’audit} (\texttt{INC-xxxx}, statuts, horodatage).
\end{itemize}

\SecRoman{Périmètre fonctionnel}
Le périmètre couvre le cycle de vie d’un incident (création, suivi, résolution), l’authentification par rôles et les statistiques agrégées. Hors périmètre : application mobile native (perspective).

\ChapterRoman{Analyse des besoins et spécifications}
\label{chap:II}

\begin{longtable}{@{}p{3.4cm} >{\RaggedRight\arraybackslash}p{10.8cm}@{}}
  \toprule
  \textbf{Fonctionnalité} & \textbf{Description} \\
  \midrule
  \endfirsthead
  \toprule
  \textbf{Fonctionnalité} & \textbf{Description} \\
  \midrule
  \endhead
  Authentification & Inscription, connexion, JWT, profil \texttt{/auth/me}. \\
  Signalement & Création \texttt{POST /incidents} en multipart (JSON + photo). \\
  Carte & Projection légère \texttt{GET /incidents/map}. \\
  Tableau de bord & Agrégats \texttt{GET /stats/dashboard}. \\
  Alertes & Création et envoi e-mail pour sévérités \texttt{HIGH} / \texttt{MEDIUM}. \\
  Administration & Gestion utilisateurs, statut incident, alertes (rôle \texttt{ADMIN}). \\
  \bottomrule
\end{longtable}

\SecRoman{Besoins non fonctionnels}
\begin{itemize}
  \item \textbf{Performance :} pagination et projections pour la carte ;
  \item \textbf{Sécurité :} mots de passe \texttt{BCrypt}, filtre JWT, règles Spring Security ;
  \item \textbf{Maintenabilité :} couches controller / service / repository, OpenAPI ;
  \item \textbf{Disponibilité :} Actuator \texttt{/health}, journalisation fichier.
\end{itemize}

\SecRoman{Règles métier notables}
Sévérités \texttt{HIGH}, \texttt{MEDIUM}, \texttt{LOW} ; alerte si \texttt{HIGH} ou \texttt{MEDIUM} ; rôles \texttt{ADMIN} / \texttt{CITIZEN}.
